# Git Workflow

For release branches, merge/tag strategy, and closeout expectations, use `docs/process/GIT_WORKFLOW.md`. This document remains the concise git baseline.

## Branches
- main: production-ready
- feature/name: new features
- fix/name: bug fixes
- docs/name: documentation-only changes

## Commits
Use clear commit messages:
- feat: add pool test history
- fix: correct finance card calculation
- docs: update roadmap
- refactor: simplify dashboard components

## Pull Requests
Every meaningful change should include:
- Summary
- Files changed
- Testing notes
- Risks
- Documentation updates
