Validation commands:

docker exec -i supabase_db_familyos psql -U postgres -d postgres

## Results

Clean local replay passed.

Supabase stopped, removed local state without backup, restarted, and replayed migrations from `supabase/migrations/` in order.

Observed migration order:

1. `20260626_baseline_schema.sql`
2. `20260627_household_foundation.sql`

The optional seed step ran after migrations and skipped starter data because `seed_user_id` still uses the placeholder value.

- Created current Family OS module tables.

Postflight passed with zero auth users, zero profiles, zero bootstrap mappings, zero active owner memberships, and zero unmapped legacy rows. That is expected after a clean reset with no local auth users.
