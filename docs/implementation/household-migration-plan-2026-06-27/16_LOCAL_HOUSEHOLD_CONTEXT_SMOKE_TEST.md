## Issues Found

### Missing Foundation Table Grants

The first household-context query failed with:

```text
permission denied for table profiles
```

Root cause:

The household migration created foundation tables and RLS policies, but did not grant table privileges to `authenticated`. Supabase RLS policies do not replace SQL table privileges.

Fix made:

Added authenticated grants for:

- `profiles`
- `households`
- `household_members`
- `people`

The same grants were applied to the local database for this smoke test.

### Missing Legacy Module Table Grants

Representative module reads initially failed with:

```text
permission denied for table tasks
```

Root cause:

The local Supabase config generated by `supabase init` does not auto-expose new public tables. The baseline schema created user-scoped module RLS policies but did not grant table privileges to `authenticated`.

Fix made:

Added `grant select, insert, update, delete` for authenticated users to each existing baseline module table, while preserving the current `user_id = auth.uid()` RLS policy.

## Dashboard/Tasks Migration Readiness

Dashboard and Tasks can be considered candidates for the next migration step, but only one module should be migrated first.

Recommended next module:

- `tasks`

Reason:

Tasks has a smaller data model than Dashboard, while Dashboard aggregates many modules and should wait until at least one source module has proven the pattern.

## Production Go/No-Go

Production migration remains no-go.

Reasons:

- Local smoke testing passed only after grant fixes that still need review.
- Local test users and bootstrap data were created manually and are not a production onboarding path.
- Feature modules still depend on legacy `user_id` queries.
- Module-table RLS has intentionally not been migrated to household-based policies.

Local-only next step is go:

- Review and commit grant fixes.
- Re-run local Supabase from a clean reset when ready.
- Begin a focused `tasks` household migration after the clean reset confirms the corrected migrations work from scratch.
