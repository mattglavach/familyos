# Product Strategy

## Source Of Truth
The permanent product handbook starts with `docs/product/PRODUCT_VISION.md`. This strategy document is a short supporting summary.

## Strategy
Build Family OS as a modular household operating system with one trusted daily dashboard and focused modules that share household identity, permissions, and design language.

## Prioritization Principles
Prioritize features that:
1. Save recurring time
2. Reduce missed responsibilities
3. Centralize scattered information
4. Support important family decisions
5. Create reusable infrastructure for future modules

## MVP Focus
The MVP and Release 1.0 should prove the core pattern:
- Dashboard
- Authentication
- Family profiles
- Tasks
- Household collaboration
- Calendar status
- Supabase persistence
- Clean mobile-first interface

## Recommended Build Order
1. App shell, navigation, authentication
2. Dashboard
3. Family members and settings
4. Tasks
5. Household collaboration
6. Release 1.0 readiness for dashboard, tasks, calendar status, household, settings, navigation, and responsive UX
7. Shopping, life lists, meal planning, home, finance, pool, college, documents, AI, and smart home expansion after the core operating loop is stable
