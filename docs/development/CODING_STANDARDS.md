# Coding Standards

## React
- Use functional components.
- Keep components small.
- Prefer composition.
- Use hooks for reusable logic.

## TypeScript
- Use explicit types for shared data models.
- Avoid `any` unless justified.
- Keep types close to the module unless shared.

## Styling
- Use Tailwind and shadcn/ui.
- Avoid one-off styling when a reusable component is better.
- Preserve design consistency.

## Data
- Centralize Supabase queries.
- Handle loading, empty, and error states.
- Validate user input.

## Comments
Use comments for why, not obvious what.
